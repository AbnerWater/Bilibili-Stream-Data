# Bilibili Stream Data

一个本地运行的 Bilibili 直播数据查询 Web 程序，用于查看登录用户在不同直播间的观看时长、弹幕数量、粉丝勋章和大航海状态，并按观看数据生成排行。

本项目默认只展示当前接口能够相对明确验证的数据。现有 `GuardActive`、粉丝勋章等接口不能可靠查询“用户在某直播间的总消费金额”，因此本项目不展示消费总额、消费金额或消费排行，避免把大航海或勋章字段误标为全量消费。

## 功能特性

- 本地 Web 界面：前端运行在 `http://127.0.0.1:5173`，后端本地代理请求 B 站接口。
- 登录态配置：输入 B 站 UID 和 Cookie 后，由后端验证并保存到内存会话。
- 粉丝灯牌扫描：根据登录用户的粉丝勋章列表自动发现主播。
- 手动添加主播：支持按主播名称搜索，也支持直接输入主播 UID 追加查询。
- 观看排行：展示主播昵称、UID、直播间号、头像、观看时长、弹幕数、勋章等级和大航海状态。
- 实时进度：扫描任务通过 SSE 推送进度、失败数和汇总信息。
- 主播详情：查看直播间链接、观看时长明细、勋章与守护状态、数据来源说明。
- 数据导出：支持 CSV、XLSX、JSON 三种导出格式。

## 技术栈

- 前端：React、Vite、TypeScript、lucide-react
- 后端：Node.js、Fastify、TypeScript
- 数据处理：xlsx、zod
- 测试：Vitest

## 快速开始

建议使用 Node.js 20+ 和 pnpm。

```bash
pnpm install
pnpm dev
```

启动后打开：

```text
http://127.0.0.1:5173
```

默认开发命令会同时启动：

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:3001`

## 使用流程

1. 登录 Bilibili 网页端账号。
2. 获取当前账号的 UID 和 Cookie。
3. 在页面顶部填写“我的 UID”和 Cookie。
4. 点击“保存登录态”，等待后端验证账号。
5. 点击“开始查询”，程序会根据粉丝灯牌扫描主播并生成排行。
6. 如需补充主播，可以在“添加主播”区域搜索主播名称，或直接输入主播 UID。
7. 在排行表中点击主播行，可在右侧查看详情。
8. 查询完成后可导出 CSV、XLSX 或 JSON。

## Cookie 获取与安全

Cookie 可以从浏览器开发者工具中复制：

1. 打开 Bilibili 网站并保持登录。
2. 打开开发者工具，进入 Network 面板。
3. 刷新页面或打开任意 B 站接口请求。
4. 在 Request Headers 中找到 `Cookie`，复制完整 Cookie 值。

安全说明：

- Cookie 只保存在本地后端内存中，不写入文件，不进入日志，不返回前端。
- 请不要把 Cookie 提交到 GitHub、截图公开发布或发送给他人。
- 如果 B 站接口返回 `code=-101`，说明登录态失效，需要重新复制 Cookie。
- 本项目适合本机使用，不建议直接暴露到公网。

## 主要数据接口

本项目通过本地后端代理访问 B 站接口，前端不会直接请求 B 站 API。

- `MedalWall`：获取用户展示的粉丝勋章墙。
- `get_home_medals`：分页补全登录用户的粉丝勋章列表。
- `GuardActive`：查询用户在指定主播直播间的观看时长、弹幕数、勋章和大航海状态。
- `live_user` 搜索：按主播名称搜索候选主播，用于手动添加。

本地后端主要接口：

- `POST /api/session`：保存并验证 UID/Cookie。
- `POST /api/query-jobs`：启动粉丝灯牌扫描任务。
- `POST /api/query-jobs/manual`：创建空的手动查询任务。
- `GET /api/query-jobs/:jobId/events`：订阅查询进度。
- `GET /api/query-jobs/:jobId`：读取查询结果。
- `GET /api/anchors/search?keyword=<name>`：搜索主播候选。
- `POST /api/query-jobs/:jobId/anchors`：追加或更新主播查询结果。
- `GET /api/query-jobs/:jobId/export?format=csv|xlsx|json`：导出结果。

## 为什么没有消费金额

当前使用的 B 站接口可以返回观看时长、弹幕数、粉丝勋章和部分大航海相关字段，但不能证明这些字段等于用户在某直播间的全量礼物消费金额。

因此，本项目不展示“消费总额”“总消费金额”“消费排行”等字段。大航海状态只作为守护状态展示，不换算为消费金额。

## 常见问题

### 提示登录态失效

如果出现“B站登录态失效”或 B 站返回 `code=-101`，请重新登录 B 站网页端并复制新的 Cookie。

### 搜索主播时提示 412 或风控

B 站搜索接口可能触发风控。可以稍后重试，或直接输入主播 UID 添加主播。

### 主播头像无法显示

前端会优先使用接口返回的主播头像，并设置 `referrerPolicy="no-referrer"`。如果图片仍加载失败，会显示文字头像作为 fallback。

### 查询结果是否会持久保存

当前查询任务和 Cookie 都保存在后端内存中。服务重启后，登录态和查询结果会丢失。
